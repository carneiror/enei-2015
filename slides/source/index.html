Rendering

  • Recalculate style
  • Layout

  • Composite layers

Recalculate Style

Matches the CSS styles with your HTML (DOM)

Runs every time a CSS property changes

1- ID, e.g. #header
		2- Class, e.g. .promo
		3- Type, e.g. div
		4- Adjacent sibling, e.g. h2 + p
		5- Child, e.g. li > ul
		6- Descendant, e.g. ul a
		7- Universal, i.e. *
		8- Attribute, e.g. [type="text"]
		9- Pseudo-classes/-elements, e.g. a:hover

Selectors ordered from more to less efficient:

Layout

Calculation of positions and geometry

Only CSS properties that change geometry and position affects this (e.g. width, height, top, margin)

Protip:  Keep your DOM as short as possible!

Paint and Setup

Fill out the pixels for each element into layers

Only affected by properties like color, border-radius, background

Takes longer when using inefficient styles

Note: This step is the heaviest from all. Modern browsers use hardware acceleration to improve the performance.

Composite Layers

Draw layers into the screen

Only affected by transform and opacity

This is VERY cheap to do

Scroll

Scroll events trigger new paints

Avoid doing anything expensive on scroll events

Protip 1: use a translateZ(0) to enable GPU acceleration

Protip 2: use "pointer-events: none" to disable hover effects during scroll  http://jsbin.com/oNiVUYe/5/quiet

Memory Leak: Nodes

You can't control when Garbage collector is called in javascript

GC won't remove your nodes until there is no reference to it.

Very easy to create leak, extremely hard to find the reason

Memory Leak: Listeners

Use an event system for attaching event handlers (e.g. jQuery, Dojo)

Always remove your listener when not needed

Avoid circular references

Event Listeners

<body>
		<div class='menu-bar' id='myMenu'></div>
		<script language='javascript'>
		var menu = document.getElementById('myMenu');
		AttachEvent(menu);
		function AttachEvent(element) {
		  element.attachEvent( "onmouseover", mouseHandler);
		  function mouseHandler(){ /* whatever */ }
		}
		</script></body>

DOM element > event handler > closure scope > DOM

 

The DOM element and the event listener will never be clear by the Garbage Collector because the circular reference.
